# 会议执行闭环 Agent
基于 OpenClaw 架构 + 飞书生态的企业级执行型 Agent，实现会议到任务执行的全流程自动化闭环。

## 🎯 项目定位
不是普通的聊天机器人，而是真正推动团队执行闭环的企业级生产力工具，完整实现：
**会议纪要 → 自动提取任务 → 自动创建飞书任务 → 自动通知负责人 → 进度跟踪 → 风险预警 → 自动周报**
全链路自动化，大幅提升团队执行效率。

## ✨ 核心特性（MVP 已实现）
- 📄 **自动读取飞书文档**：支持新版/旧版飞书会议纪要自动读取
- 🤖 **结构化任务提取**：基于 LLM 结构化输出能力，精准提取任务、负责人、截止时间、优先级
- ✅ **自动创建飞书任务**：直接打通飞书任务系统，自动创建标准化任务
- 💬 **智能通知推送**：自动给负责人发送任务卡片通知，支持直接跳转任务详情
- ⚡ **高可扩展架构**：完全遵循 OpenClaw 设计规范，模块低耦合，能力可插拔
- 🛡️ **企业级健壮性**：内置重试、异常处理、参数校验，生产环境可用

## 🚀 快速开始
### 1. 安装依赖
```bash
pip install -r requirements.txt
```

### 2. 配置环境变量
```bash
cp .env.example .env
```
编辑 `.env` 文件，配置以下必要参数：
- 飞书开放平台 `FEISHU_APP_ID`、`FEISHU_APP_SECRET`
- LLM 服务 `LLM_API_KEY`、`LLM_BASE_URL`、`LLM_MODEL_NAME`

### 3. 本地运行
```bash
cd app
python main.py
```

### 4. 接口测试
服务启动后访问 http://localhost:8000/docs 查看 Swagger 接口文档，调用 `/process_meeting` 接口测试流程：
```json
{
  "meeting_doc_id": "你的飞书文档ID",
  "doc_type": "docx"
}
```

## 🏗️ 项目架构
完全遵循 OpenClaw 架构规范，分层设计，高可扩展：
```
meeting-execution-agent/
├── app/
│   ├── agents/          # Agent 编排层：核心工作流调度，仅做决策，不实现能力
│   ├── tools/           # 工具层：所有原子能力实现，可被 Agent 灵活调用
│   ├── prompts/         # Prompt 层：统一管理 LLM 提示词
│   ├── services/        # 服务层：封装第三方通用能力（飞书客户端、LLM 客户端等）
│   ├── config/          # 配置层：环境变量管理、全局配置
│   ├── models/          # 模型层：全链路结构化数据定义，类型校验
│   ├── scheduler/       # 定时任务层：后续实现任务巡检、预警、周报生成
│   └── main.py          # FastAPI 服务入口
```

## 📚 API 文档
### 核心接口
- `POST /process_meeting` - 处理会议纪要，执行完整闭环流程
- `GET /health` - 健康检查接口

详细接口文档请访问服务启动后的 `/docs` 路径。

## 🔮 后续扩展规划（MVP 后实现）
